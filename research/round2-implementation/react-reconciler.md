# Reconciler Research Digest — What ReactiveUI-Unreal's C++ Reconciler Should Copy From React (Beyond the Godot Port)

Sources examined: `addons/reactive_ui/core/{reconciler.gd, fiber.gd, hooks.gd, vnode.gd, v.gd, host_config.gd, context.gd, suspense.gd, component_state.gd, config.gd}` (absolute root: `c:\Yanivs\GameDev\ReactiveUI\ReactiveUI-Gadot\addons\reactive_ui\core\`), plus React 18/19 fiber internals (web-verified where noted).

**CRITICAL CORRECTION to the assignment premise:** the Godot port NO LONGER rebuilds fresh fibers each pass. Commit `f300ee3` ("perf(core): reuse_by_slot, node pool, mark-sweep keyed diff") added full React-style double-buffering: `_reconcile()` reuses `old_fiber.alternate` as the WIP fiber (`[perf #1]` comments), `_begin_render()` ping-pongs the root's alternate, and `_commit_root()` swaps `_root_current = _wip_root` with no per-frame tree-sever. The doc-comment at `reconciler.gd:21-22` ("Fresh fibers each pass — no 2-object double-buffer reuse") is **STALE** and contradicts the code; do not propagate it into the Unreal plan.

## 1. Inventory — what the Godot port implements today

### Work loop (`reconciler.gd`)
- **Synchronous by default**; optional time-slicing via `RUIConfig.time_slicing` (default `false`) + `RUIConfig.frame_budget_ms` (default `8.0`) — render phase parks between frames (`process_frame` `CONNECT_ONE_SHOT`), **commit stays atomic**. Initial mount always synchronous (no empty first frame).
- `_perform_unit(fiber)` = inlined begin_work: `Tag.FUNCTION → _begin_function`, `Tag.ERROR_BOUNDARY → _begin_error_boundary`, else (ROOT/HOST/FRAGMENT/PORTAL) reconcile declared children. Leaf fast-path: no declared children + no old children → skip reconcile entirely. Then `_complete_work` on the way up (sibling → parent climb).
- **Update coalescing:** hook setter → `schedule_update_on_fiber` → marks `fiber.has_pending_update` + ancestors' `subtree_has_updates` → `call_deferred("_tick")` guarded by `_tick_pending` = one re-render per frame. setState **during commit** → `_deferred_updates` replayed after commit. setState **mid-render** → `_restart` = restart-from-root next tick (clears effect list; invariant against stale Placement effects); 25-restart guard with `push_error`.

### Fiber (`fiber.gd`)
- Tags: `FUNCTION, HOST, FRAGMENT, PORTAL, ERROR_BOUNDARY, ROOT`. Effect flags bitmask: `EFFECT_PLACEMENT=1, EFFECT_UPDATE=2, EFFECT_DELETION=4, EFFECT_LAYOUT=8, EFFECT_PASSIVE=16, EFFECT_PORTAL_RETARGET=32`.
- Fields: `parent/child/sibling/index` (linked tree), `key`, `type` (host class name), `component: Callable`, `props` (committed) / `pending_props` / `input_children`, `node`, `portal_target`, `alternate`, `effect_tag`, `next_effect` (singly-linked post-order effect list), `deletions: Array` per-parent, `matched_pass` (transient mark-sweep flag, GO-08), `provided_context`, `reads_context`, `has_pending_update`, `subtree_has_updates`, `state` (RUIComponentState, **shared across alternates**), inline apply-plan cache (`apply_size/apply_special/apply_plain`), error-boundary fields (`eb_active/eb_showing_fallback/eb_last_error/eb_reset_key/eb_fallback/eb_handler/eb_children`).
- `matches(vnode)`: same kind + (HOST: type equal | FUNCTION: `component == vnode.component` Callable equality).

### Double buffering
- `_reconcile`: type-match → reuse `old_fiber.alternate` (alloc only on true placement/type change). Reused fiber carries committed baseline: `node, state, props, reads_context, has_pending_update, subtree_has_updates`, `provided_context.duplicate()`, apply plan, eb flags. Render-scoped fields reset every pass. `RUIComponentState` is reference-shared across alternates; `state.fiber` re-pointed to the live fiber in `_begin_function`.
- `_release()` on deletion/unmount severs RefCounted cycles for the subtree **and its alternate buddies** (GDScript ref-cycle GC workaround).

### Child reconciliation — three tiers (`_reconcile_children`)
1. **`_try_fast_leaf_list`**: stable list of **childless host leaves** (same count/type/key per slot on both sides) → reuse child fibers **in place** (no buddy swap, no per-child begin/complete), diff props per row, effect-list only changed rows. **GO-09 `reuse_by_slot`** container prop: same path but **ignores key churn** — the node at slot *i* is reused for whatever leaf lands at slot *i* (pure props UPDATE, zero add/remove/free); contract: childless host leaves with no identity to preserve (no ref/focus/caret/scroll/selection/animation).
2. **`_keys_stable`** pre-pass: positionally identical keyed list → in-place reconcile, no key map, no reorder.
3. **Full keyed mark-sweep (GO-08)**: persistent `_key_map` Dictionary (cleared per call, no per-frame alloc) mapping `_fiber_key` (user key or namespaced positional `"\x01idx%d"`) → old fiber; per-fiber `matched_pass` bool instead of a `matched` dict; trailing sweep deletes unmatched. Mirrors Unity's `[ThreadStatic] s_childMap`. Unkeyed path = positional walk with type-match. **No minimal-move computation**: any placement/move/deletion sets `structural` → `_mark_reorder(parent)` → commit-time `_enforce_child_order` does a full O(n) `move_child(nd, i)` assert of the flattened host-child order (`_collect_host_children` flattens through fragments/components, skips portals).

### Bailout (`_begin_function`)
`can_bail = !has_pending_update AND context_ok AND props_equal AND children_same` where props_equal = `is_same()` identity fast-path → optional `props.__memo_eq` custom comparer (`V.memo`) → deep `==`; context_ok re-resolves every recorded context dep vs its recorded value; children compared by vnode-list equality. On bail: **reuses `state.last_output` (cached vnodes) without re-running the render fn — but still calls `_reconcile_children` on it, i.e. the whole subtree is still walked/re-paired every pass.** `subtree_has_updates` is *set* everywhere but never *consumed* to skip descent — the header explicitly says "true O(changed) subtree carry-over is a later perf optimization (see PORTING_PLAN 1.9)". **This is the single biggest React mechanism NOT yet copied.**

### Commit phase (`_commit_root`) sub-phases, in order
1. All deletions (`_commit_deletion`: null refs recursively → run all effect cleanups recursively → free-or-pool host nodes → `_release` fibers). 
2. Walk effect list once: `PLACEMENT` (add_child under `_host_parent_node`, honoring portals + `resolve_child_host`), `UPDATE` (`_commit_update` with **inline apply-plan cache**: props classified once into plain keys vs "special" [events/ref/style/items/classes — sticky]; plain elements get a lean diff+write via `RUIHost._set_prop`, special elements the generic `RUIHost.apply_props`), `PORTAL_RETARGET`, `LAYOUT` (layout effects, cleanup-then-setup per fiber, deps-gated), collect `PASSIVE`. 
3. `_reorder_set` → `_enforce_child_order` per marked host/portal parent. 
4. Swap current↔WIP. 
5. `_flush_passive`: **two passes across all fibers — every cleanup first, then every setup** (deps-gated; `deps==null` every commit, `[]` mount-only). Synchronous, same frame. 
6. Replay `_deferred_updates`.

### GO-05 host-node pool
Per-reconciler `_node_pool: Dictionary` class→Array, `_POOL_CAP_PER_CLASS := 256`, gated by `RUIConfig.host_node_pool` (default on). Deleted **childless leaf** nodes are detached (not freed) with last props stashed under `__rui_pool_old` meta; `_acquire_node` reuses via `apply_props(node, pool_old, new_props)` diff + `reset_removed_plain` (ClassDB per-class default cache) — cheaper than instantiate+full-apply for homogeneous churn. Item-model controls (`items` prop) never pooled. Pool drained on `unmount()`.

### Context (`hooks.gd` + `context.gd`)
- `RUIContext` handle: object-identity map key + `default` value; String keys back-compat. `useContext` consumes **no hook slot** (records `context_deps` out-of-band so it can't perturb hook order); resolution = **O(depth) parent-chain walk** per read (`_resolve_context`). `provideContext(key, value)` is a **hook-style call inside the provider component** (no `<Provider>` wrapper element — family divergence from React); on value change it **eagerly propagates**: DFS over the committed alternate subtree (`_propagate_context_change`) marking consumers `has_pending_update` and intermediate ancestors `subtree_has_updates`, stopping at shadowing providers.

### Hook state storage (`component_state.gd`)
Positional `hooks: Array` of Dictionary slots + **separate cursors** for state hooks (`hook_index`), passive effects (`effect_index`), layout effects (`layout_index`). Setter **mutates the slot value in place immediately** — no update queue; equal-value set bails with `_ref_equal` (Object.is semantics: identity for Array/Dictionary/Object, value for scalars). Setter/dispatch identity stable (closures capturing `(state, i)`); late calls after teardown guarded by slot-count check. Deps comparison `_deps_changed` uses **value equality (deep `==`)** — a documented deliberate divergence from React's `Object.is`-per-item ([G-09] comment). Dev diagnostics: hook-order signature validation (`hook_log` vs primed `hook_signatures`, `RUIConfig.enable_hook_validation`, debug-only) + set-during-render strict warning.

### Documented divergences (preserve-or-improve decisions for C++)
- **Structural error boundaries** — no try/catch in GDScript; `eb_active` toggled imperatively, `reset_key` clears; cannot auto-catch a child render crash.
- **`useTransition` synchronous no-op** (`is_pending` always false, action runs immediately); **`useDeferredValue`** returns previous value then commits target on a next-frame one-shot tick through the normal schedule.
- **Removed plain props not reset to defaults** (audit #23) — except on pool reuse.
- **Refs invoked every commit** (`apply_props` step 3, "so the latest node is exposed") — React only calls on attach/detach; refs nulled recursively on deletion/unmount (`_null_refs_recursive`).
- **Suspense** (`suspense.gd`) is a plain function component (no throw-to-suspend): `fallback` + `ready_signal` (awaited Signal) or `is_ready` (per-frame poll).
- **HMR/Fast Refresh**: static `_hmr_live` WeakRef registry; `hmr_refresh_all(scripts, resets, global_rerender)` marks matching FUNCTION fibers dirty (defeating cached-output bailout), resets state where hook signature changed (`_hmr_reset_state` = cleanups + dispose), then flushes synchronously unsliced. Component identity survives reload because Godot in-place-reloads the script resource so the `Callable` keeps dispatching new code (see `V.comp(path)` lazy path-based resolver + `_comp_cache`).

## 2. React 18/19 fiber internals — reference facts

- **Work loop** (`ReactFiberWorkLoop`): `renderRootSync/renderRootConcurrent` → `workLoopSync { performUnitOfWork }` → `beginWork` (descend) → `completeUnitOfWork/completeWork` (ascend). Commit = three sub-phases: `commitBeforeMutationEffects` (snapshot), `commitMutationEffects` (deletions-per-parent first, then Placement/Update via `recursivelyTraverseMutationEffects` gated by `subtreeFlags`), `commitLayoutEffects` (layout effects + `commitAttachRef`); **passive effects flushed asynchronously after paint** (`flushPassiveEffects` via Scheduler), destroy-all-then-create-all ordering.
- **Double buffering**: `createWorkInProgress(current, pendingProps)` reuses `alternate`; resets `flags/subtreeFlags`, `deletions=null`; copies `child, memoizedProps, memoizedState, updateQueue`, clones `dependencies {lanes, firstContext}`, copies `sibling/index/ref`.
- **Flags**: `Placement, Update, ChildDeletion, ContentReset, Callback, DidCapture, ForceClientRender, Ref, Snapshot, Passive, Visibility, StoreConsistency` + masks `MutationMask/LayoutMask/PassiveMask` + `RefStatic/LayoutStatic/PassiveStatic`. **React 18 removed the React-≤17 `firstEffect/nextEffect` linked effect list**; commit now traverses the tree pruned by `subtreeFlags`, bubbled in `completeWork` via `bubbleProperties` (`subtreeFlags |= child.subtreeFlags | child.flags`; `childLanes` merged the same way). (Verified.)
- **Child reconciliation** (`ReactChildFiber.reconcileChildrenArray`): **Pass 1** — same-index walk with `updateSlot` (key match → `useFiber` reuse or create-by-type; key mismatch → break). If new list exhausted → `deleteRemainingChildren`; if old exhausted → create rest. **Pass 2** — `mapRemainingChildren` into a Map keyed by `key ?? index`, `updateFromMap` per remaining element, matches removed from map, leftovers deleted. **`placeChild` `lastPlacedIndex`**: reused fiber with `oldIndex < lastPlacedIndex` gets `Placement` (move); else `lastPlacedIndex = oldIndex` — minimizes moves for right-shifting stable subsequences (LIS-lite, not true LIS).
- **Bailout**: `beginWork` — if `oldProps === newProps` and no update lanes → `attemptEarlyBailoutIfNoScheduledUpdate` → `bailoutOnAlreadyFinishedWork`: **if `!includesSomeLane(renderLanes, childLanes)` return `null` — the entire subtree is skipped**; else `cloneChildFibers(workInProgress)` (clone one level of children WIP, don't re-render). Function components additionally: after `renderWithHooks`, if `!didReceiveUpdate` → `bailoutHooks` + same bailout.
- **Lanes**: 31-bit bitmask — `SyncLane, InputContinuousLane, DefaultLane, TransitionLanes, RetryLanes, IdleLane, OffscreenLane`; React 19 adds a `DeferredLane` for `useDeferredValue`.
- **Context** (`ReactFiberNewContext`): `pushProvider/popProvider` on a global `valueCursor` stack; the context object holds `_currentValue` directly → **O(1) read**, no ancestor walk; `readContext` appends to `fiber.dependencies.firstContext`. **Lazy context propagation shipped in React 19** (PR #20890, flag cleanup PR #31810): the subtree scan is deferred until a bailout would occur; all changed contexts checked simultaneously via `checkIfContextChanged` over the dependencies list; eager `propagateContextChange_eager` was the pre-19 path. (Verified.)
- **Hook state**: linked list on `fiber.memoizedState` — `Hook {memoizedState, baseState, baseQueue, queue, next}`; per-state-hook **circular update queue** of `Update {lane, action, eagerState}`; `dispatchSetState` does **eager state computation** (bail without scheduling when `Object.is(eagerState, currentState)` and queue empty). Hook list is **double-buffered** (`updateWorkInProgressHook` clones onto WIP) — an aborted render discards WIP hooks, current untouched.
- **Refs**: `commitAttachRef` in layout phase after mutation; `safelyDetachRef` in mutation phase (on deletion and when ref identity changes). React 19: **ref callbacks may return a cleanup function** (`refCleanup`); ref-as-prop.
- **Error boundaries**: render throw → `handleError`/`throwException` walks the return path to the nearest class boundary (`getDerivedStateFromError`/`componentDidCatch`) → `ShouldCapture` → `unwindWork` **pops context/host stacks** → boundary re-renders with `DidCapture` showing fallback. Exception-dependent.
- **`useSyncExternalStore`**: render calls `getSnapshot()` and stores `{value, getSnapshot}`; **subscription happens in an effect** (`subscribeToStore`), which re-checks the snapshot on subscribe (tear window between render and effect) via `checkIfSnapshotChanged` → `forceStoreRerender` on SyncLane; concurrent renders push a `StoreConsistency` check before commit.
- **StrictMode (dev)**: double-invoke render functions (discard first), double-invoke useState/useReducer initializers, and (18+) effect mount→unmount→remount simulation.
- **HostConfig** (react-reconciler mutation mode): `createInstance(type, props, rootContainer, hostContext, internalHandle)`, `createTextInstance`, `shouldSetTextContent`, `appendInitialChild`, `finalizeInitialChildren` (→ `commitMount`), `getRootHostContext/getChildHostContext`, `prepareForCommit/resetAfterCommit`, `appendChild/appendChildToContainer`, `insertBefore/insertInContainerBefore`, `removeChild/removeChildFromContainer`, `clearContainer`, `commitUpdate`, `commitTextUpdate`, `hideInstance/unhideInstance` (Suspense), `getPublicInstance`, `detachDeletedInstance`, `scheduleMicrotask`. **React 19 removed `prepareUpdate`/updatePayload — `commitUpdate(instance, type, oldProps, newProps, internalHandle)` now diffs directly in commit.** (Verified; this broke react-three-fiber and other custom renderers.)

## 3. Gap table: React mechanism → Godot port status → C++ (Unreal/Slate) recommendation

| React mechanism | Godot port status | C++ recommendation + rationale |
|---|---|---|
| **Fiber double-buffering / alternate reuse** | **DONE** (since f300ee3): `_reconcile` reuses `old_fiber.alternate`; zero fiber alloc for stable positions; state shared across buddies; stale header comment says otherwise | **Copy React/Godot: double-buffer.** In C++: fibers from a **slab allocator with free-list** (chunked arena, fiber pairs allocated together); alternate reuse = zero steady-state allocation + **pointer-stable fibers** (closures/setters hold fiber or state pointers — stability is mandatory without a GC). Fresh-per-pass would be strictly wrong in C++: alloc churn every frame + dangling-pointer hazard. `FRuiComponentState` lives outside the slab (`TSharedRef`, shared across alternates, explicit dispose on unmount to break `TFunction` capture cycles — port `_dispose_fiber_state`). |
| **`bailoutOnAlreadyFinishedWork` + `childLanes` subtree skip + `cloneChildFibers`** | **MISSING — the biggest gap.** Bailout reuses `state.last_output` but still runs `_reconcile_children` over the whole subtree every pass; `subtree_has_updates` is written but never consumed (self-documented as PORTING_PLAN 1.9 future work) | **Copy from React — this is the "copy more from there" headline.** Bubble a dirty mask up (`childLanes`-style, `subtree_has_updates` consumed for real): in begin_work, if props identical + no pending update + no context change AND no dirty descendant → **return null, skipping the subtree entirely** (WIP adopts the current children via one-level `cloneChildFibers`). Makes a 60 Hz HUD re-render O(dirty path) instead of O(tree). Prerequisite: context-change propagation must mark consumers dirty (Godot already has `_propagate_context_change` — port it; the two compose exactly as in React). Keep Godot's fast-leaf-list/`_keys_stable`/`reuse_by_slot` as additional tiers — they attack list-churn, which subtree-skip doesn't. |
| **effectTag/flags bitmask + `subtreeFlags`** | Flags bitmask done (6 flags, incl. custom `EFFECT_PORTAL_RETARGET`); **React-17-style singly-linked effect list** (`next_effect`, built in complete_work + fast-list path); no `subtreeFlags` | Keep the **flags bitmask** (uint32, room for `Ref`, `Snapshot`, `Visibility` later). For a synchronous renderer the **17-style effect list is the right choice** — O(changed) commit, one pointer per fiber, trivially correct without Suspense/Offscreen re-entrancy (the reason React dropped it). Add `subtreeFlags`-style bubbling **only** as the dirty-mask for the subtree-skip bailout above (dual-purpose field), not for commit traversal. |
| **Deletion tracking** (`parent.deletions` + `ChildDeletion`) | Done: per-parent `deletions` array + global `_deletions`, `EFFECT_DELETION`, all deletions committed before the effect list; deletion runs refs-null → cleanups → free/pool → `_release` | Copy Godot's simpler shape (global deletion list, flush first). In Slate, deletion = remove from parent slot (`RemoveSlot`/`ClearContent`) then drop the `TSharedRef` — the widget frees when the last ref dies (no `queue_free`). **`detachDeletedInstance` matters**: refs/pool entries must not keep widgets alive; hand out `TWeakPtr<SWidget>`. |
| **Child reconciliation algorithm** (two-pass: same-index walk → map; `placeChild` `lastPlacedIndex` minimal moves) | Different: tiered fast paths + **full-map mark-sweep** (`_key_map` + `matched_pass`) with **no move minimization** — commit-time `_enforce_child_order` re-asserts full order via `move_child(nd, i)` | **Copy React's two-pass + `lastPlacedIndex`, and drop enforce-order as the primary mechanism.** Rationale: Godot's `move_child` is cheap; Slate slot manipulation is not — each `RemoveSlot`/`InsertSlot` triggers layout invalidation (`Invalidate(EInvalidateWidgetReason::ChildOrder)`), so the O(n) order-assert per structural frame is a real cost. React's first-pass same-index walk also beats always-building the map for the common prefix-stable case (Godot compensates with `_keys_stable`; React's pass 1 subsumes it). Map to Slate as `InsertSlot(index)`/`RemoveSlot(widget)` per moved child only. **Keep** Godot's fast-leaf-list + `reuse_by_slot` as tier-0 (huge for game lists; React has no equivalent), and keep the namespaced-positional-key trick (`"idx%d"` control-char-prefixed) for mixed keyed/unkeyed lists [audit M1]. Fallback: for panels without indexed insert, keep an enforce-order pass per-panel-adapter. |
| **Lanes / priority** | Two effective priorities: sync (mount/HMR flush) + one coalesced deferred render per frame; `useDeferredValue` = ad-hoc next-frame tick; restart-from-root on mid-render update; optional frame-budget time-slicing (restart semantics, atomic commit) | **Do NOT port the 31-lane model.** Minimal subset: a 3-value lane enum in a uint32 — `SyncLane` (mount, `FlushSync`, HMR), `DefaultLane` (coalesced per-frame), `DeferredLane` (React-19-style, backs `useDeferredValue` so it stops being a hook-local hack). Keep Godot's restart-from-root + 25-restart guard + atomic commit + optional frame-budget slicing (park/resume on Slate tick). Reserve bits for a future TransitionLane; never implement entanglement. |
| **Context: push/pop value stack, O(1) reads** | Handle-based (`RUIContext`, object identity + default) but **O(depth) parent walk per `useContext` read**; `provided_context` Dictionary per fiber; hook-order-safe out-of-band dep recording | **Copy React's cursor stack**: `TRuiContext<T>` holds a value stack; provider fiber pushes in begin_work, pops in complete_work **and in every unwind/bailout-skip path** (the classic bug source — the subtree-skip bailout returns null past providers, so pop bookkeeping must live in the ascend loop). O(1) typed reads, no walk. Keep the handle+default+object-identity design and the out-of-band `context_deps` (no hook slot) — both are correct and better than React's `<Provider>` wrapper for the family's `provideContext`-as-a-call API. |
| **Context propagation (eager vs React-19 lazy)** | **Eager** DFS over committed subtree marking consumers `has_pending_update` + ancestors `subtree_has_updates`, stopping at shadowing providers — this is pre-18 React's `propagateContextChange_eager`, already correct | Port Godot's eager propagation first (it is what makes subtree-skip bailout safe). React 19's lazy propagation (defer scan until a bailout would occur, check all contexts at once via the fiber's dependencies list) is a **later optimization** — note it in the plan as the upgrade path if provider-heavy trees profile badly; it requires the `dependencies.firstContext` list per fiber, which the out-of-band `context_deps` already approximates. |
| **memoizedProps/pendingProps bailout** | Done: `props` vs `pending_props`, `is_same()` identity fast-path → `__memo_eq` custom comparer (`V.memo`) → deep `==`; children compared by vnode-list identity | Copy, but the C++ props representation decides everything: recommend `FRuiProps` = small flat array of `(FName, FRuiValue)` pairs behind `TSharedRef<const FRuiProps>` so **identity compare (pointer) is the memo fast-path** and shallow per-key compare the fallback. `FRuiValue` = `TVariant` of blessed types (bool/int64/double/FString/FName/FText/FLinearColor/FMargin/FVector2D/TSharedPtr\<void\>/TFunction-wrapper/…). Keep `__memo_eq` as an optional comparer on the vnode. **Warning:** `TFunction` has **no operator==** — see the event-handler row below; this quietly breaks Godot's `old_cb != new_cb` event-diff and the `component ==` fiber match. |
| **Component identity for `matches()`** | `Callable` equality (`component == vnode.component`); survives Godot in-place script hot-reload | **Cannot use raw function pointers in C++**: UE Live Coding relocates code, breaking identity across reload (destroys all state, exactly what the HMR gate must avoid). Copy the spirit of `V.comp(path)`: a **component registry** — components register a stable `FName` id (the `.uetkx` codegen emits it); the vnode stores the id; `matches()` compares ids; the registry's fn pointer is re-bound after Live Coding; a hook-signature hash decides HMR state reset (port `_hmr_reset_state` / Unity `FullResetComponentState`). |
| **Hook state: update queue + eager bail + double-buffered hook list** | Divergent-but-working: slot value mutated **in place immediately**, state shared across alternates, no queue; equal-set bails via `_ref_equal` (Object.is semantics); safe only because restart-from-root re-reads latest values | For a synchronous renderer, **keep the Godot model** (direct-mutate + Object.is bail + shared state) — the update queue exists for interleaved lanes we don't have; document that render-phase setState = restart (keep the guard + `enable_strict_diagnostics` warning). Copy React's **eager bailout** (already present as `_ref_equal` check — keep). If real time-slicing with non-restart resumption ever ships, revisit with a queue. Storage: positional slot array with **separate cursors for state/passive/layout** (Godot's split is good — keep), each slot a small-buffer type-erased cell (placement-new + destroy/equals fn pointers) rather than React's linked list (cache-friendlier, no per-hook alloc). |
| **Refs lifecycle** | Divergent: ref invoked **every commit**; nulled recursively on deletion/unmount; box `{current}` or Callable | **Copy React, not Godot**: attach in layout phase after placement; detach (call with null / run cleanup) on deletion **and on ref-identity change**; adopt **React 19 cleanup-returning ref callbacks** (natural in C++: `TFunction<TFunction<void()>(TWeakPtr<SWidget>)>`). Ref value = `TWeakPtr<SWidget>`, never `TSharedRef` (must not extend widget lifetime; also protects the node pool). Every-commit invocation is wasted work and violates user expectations ported from React. |
| **Error boundaries (throw → `ShouldCapture` → unwind)** | Structural only: `eb_active` imperative toggle, `reset_key`, `fallback`, `on_error`; documented no-auto-catch | **UE ships with C++ exceptions disabled by default (`bEnableExceptions=false`)** — the React throw path is unavailable in shipping. Recommend: (1) keep structural boundaries (family parity: `eb_active/reset_key/fallback/on_error`); (2) add a **cooperative error channel**: render returns `TValueOrError<FVNodes, FRuiError>` or sets a thread_local `FRuiRenderError` latch (`RUI_RENDER_FAIL(...)` macro) checked after each component render → reconciler **unwinds the WIP to the nearest boundary, popping context stacks** (React's unwindWork, minus the throw), discards the partial subtree (safe pre-commit thanks to double buffering — current tree intact), renders fallback with `DidCapture` semantics. (3) Optional dev-only: editor modules may enable exceptions per-module — wrap render in try/catch there as a bonus auto-catch, never in shipping. Never `setjmp/longjmp` (skips destructors). |
| **Portals** | Done: `PORTAL` tag + `portal_target: Node`, `EFFECT_PORTAL_RETARGET` on target change + reorder at new target; host-parent resolution and child-collection honor portals | Copy as-is. Slate targets: an `SOverlay` slot handle, `SWindow::AddOverlaySlot`, or any registered panel handle; portal target as an opaque `FRuiPortalHandle` (weak widget + adapter). Retarget effect + reorder-at-target ports directly. |
| **`useSyncExternalStore` semantics (→ RUISignals port)** | `useSignal` **subscribes during render** (first call), re-binds selector/comparer every render, resubscribes on signal-instance change, unsubs via slot scan in `_dispose_fiber_state`. Hazard: a subscription created in a render pass that restarts before commit (fresh fiber, `alternate == null`) is dropped without `_release` → **leaked subscription** — works in practice, unsound in principle | **Copy React's contract**: render reads snapshot (selector applied) and stores `{value, getSnapshot}`; **subscribe in a passive effect**; on subscribe, re-check snapshot (tear window) and on store fire compare-then-schedule (SyncLane re-render). Eliminates the leaked-subscription window and is restart-safe by construction. `FRuiSignal<T>` store: `Subscribe(TFunction<void()>) → FRuiUnsubscribe`; keyed registry (`RUISignals.get_or_create` → `FName`-keyed, type-checked). **Bonus integration (project pitch alignment):** the same hook shape adapts UE's `INotifyFieldValueChanged` (FieldNotify/MVVM viewmodels) and dynamic multicast delegates as external stores — `UseFieldNotify(ViewModel, FieldId)` — which is exactly how "Epic's stack feeds us data". |
| **StrictMode dev double-render** | Absent; instead has hook-order validation (`hook_log`/`hook_signatures`) + set-during-render warning, debug-only via `RUIConfig` | **Worth it in C++ dev builds — yes, cheaply.** CVar `rui.StrictMode` (editor/dev only): double-invoke render fns discarding the first output (catches impure renders and **stale-lambda-capture bugs, which are deadlier in C++ than in GC languages**); optionally double-invoke state initializers. Effect mount→unmount→remount simulation: optional, second priority. **Also port Godot's hook-order signature validation** — it's orthogonal, cheap, and catches conditional hooks that C++ templates cannot statically prevent. |
| **HostConfig contract shape** | `RUIHost` static class: `create_node` (ClassDB by name), `apply_props(node, old, new)` (event transition + ref + style + plain diff), `_set_prop`, `resolve_child_host` (Godot lacks contentContainer indirection; `rui_content` meta opt-in), `reset_for_pool/reset_removed_plain` (pool), `warn_capacity`; reconciler calls `add_child/move_child/remove_child/queue_free` directly | **Formalize the seam React-style — this is where Slate demands more structure than Godot.** Define `IRuiHostConfig` (or a compile-time policy): `CreateInstance(FName type, const FRuiProps&) → TSharedRef<SWidget>` via a **widget registry** (Slate has no ClassDB — `FName → factory TFunction`; neither Godot nor React has this split — plan for a `slot.*` prop namespace); (b) single-content widgets — `SBorder/SBox/SButton/SScaleBox` → `SetContent/ClearContent` (this **is** Unity's contentContainer / Godot's `rui_content`, but mandatory, not opt-in — port `resolve_child_host` as a first-class adapter concept); Port **GO-05 node pooling** (childless leaves, per-type buckets, cap 256, last-props stored in the pool entry struct instead of node meta, diff-on-reuse + removed-plain reset) — Slate widget construction (SNew + style resolution) is costlier than Godot instantiate, so pooling pays MORE here. Port **GO-09 `reuse_by_slot`** verbatim. |
| **Event handlers (React `onXxx` → host events)** | `RUIHost`: camelCase `onClick`→signal aliases + polymorphic `onChange` + native `on_<signal>` escape hatch; change-detect via `Callable` equality; disconnect-on-removal | Slate events are **construction-time delegates** (`SButton::FArguments::OnClicked`) — most have post-construction setters (`SetOnClicked`) but not all; the adapter prop table must cover this. **`TFunction` has no equality** → cannot diff handlers like Godot. Recommended: reconciler installs **one stable trampoline per (widget, event) once** (a small forwarder holding a rebindable inner `TFunction`); prop updates just swap the inner target — no reconnect, no comparison needed. This is Godot's `useStableCallback` pattern promoted into the host config, and it should be the codegen default for `.uetkx` event props. |
| **Update coalescing / scheduling** | `call_deferred` one `_tick` per frame; deferred-during-commit replay; park/resume on `process_frame` | Coalesce onto the Slate/engine tick: `FSlateApplication::OnPreTick` (or `FTSTicker::GetCoreTicker`) — one render+commit before paint so state set this frame is visible this frame. `FlushSync` entry for HMR/tests. Keep `_deferred_updates` replay and the mid-render `_restart` invariant exactly. |
| **Passive effect timing** | Synchronous at end of commit (two-pass: all cleanups, then all setups), same frame | Two options; recommend **keep Godot's synchronous flush** initially (deterministic, test-friendly, no re-entrancy) and note React's post-paint deferral (queue to next ticker tick) as an opt-in `RUIConfig`-style CVar if commit-time stalls appear in profiles. Preserve the two-pass cleanup-then-setup ordering — user code depends on it. |

| **HMR / Fast Refresh** | `_hmr_live` WeakRef registry, `hmr_refresh_all`, per-fiber dirty-mark defeating cached-output bailout, hook-signature-change state reset, synchronous unsliced flush | Port the registry + synchronous flush + signature-reset logic; identity via the component FName registry (above). ## 4. The 23 hooks (`hooks.gd`) — full enumeration + C++ trickiness flags

Count = 23 as shipped: 21 `use*` + `createContext` + `provideContext`. (Router adds +17 more on `RUIRouter` — separate subsystem, out of this file's scope but the plan must not forget them.)

| # | Hook | Semantics (Godot) | C++ trickiness |
|---|---|---|---|
| 1 | `useState(initial) -> [value, setter]` | setter accepts value **or** updater `(old)->new`; stable identity; equal-set bails (`_ref_equal` = Object.is) | **Type erasure**: `UseState<T>` template → slot stores T via small-buffer placement-new + destroy/equals fn ptrs (or `TUniquePtr<IHookSlot>` per slot — simpler, one alloc). Setter = small handle `{TWeakPtr<FRuiComponentState>, int32 slot}` with `operator()(T)` and `operator()(TFunction<T(const T&)>)`; must survive teardown (slot-count guard, port [audit C3]). |
| 2 | `useReducer(reducer, initial) -> [state, dispatch]` | dispatch stable; reducer closure refreshed each render | Same erasure; reducer stored as `TFunction<T(const T&, const A&)>` — two type params → template pair `UseReducer<T, A>`. |
| 3 | `useRef(initial) -> {current}` | stable box, never re-created, no re-render on mutate | `TSharedRef<TRuiRef<T>>` box; for widget refs specifically `TRuiRef<TWeakPtr<SWidget>>`. Easy. |
| 4 | `useMemo(factory, deps)` | recompute on shallow deps change; **deps compared by VALUE (deep ==), deliberate [G-09] divergence from React's Object.is** | Deps without GC: `FRuiDeps = TArray<FRuiDepValue>` (TVariant of value types + `const void*`/`TWeakPtr` identity for objects). C++ variant equality naturally = value-equality for values, pointer for refs — **lands between Godot and React; pick and document one** (recommend: value for values, identity for shared refs = React-like, and note the [G-09] rationale for why Godot chose deep-compare — silently changing skip-behavior breaks effects that rely on equal-but-recreated deps). |
| 5 | `useCallback(cb, deps)` | = `useMemo(() => cb, deps)` | Returns cached `TFunction` — fine, but reminds: **no TFunction equality**, so downstream prop-diffs on callbacks need the identity of the *cached object* (works if props hold `TSharedRef`-wrapped functions). |
| 6 | `useImperativeHandle(factory, deps)` | = useMemo alias | Trivial. |
| 7 | `useEffect(effect, deps)` | passive, post-commit, two-pass cleanup/setup; `deps==null` every commit, `[]` mount-once; may return cleanup | **Lambda capture lifetime is the #1 footgun**: effects stored in state outlive the render stack; enforce capture-by-value convention; `TWeakObjectPtr` for UObjects, `TWeakPtr` for widgets; document loudly + StrictMode double-render to flush stale captures. Signature `TFunction<TFunction<void()>()>` with an overload returning void. |
| 8 | `useLayoutEffect(effect, deps)` | synchronous during commit, pre-paint, per-fiber cleanup-then-setup | Same as above; runs before Slate paint (commit inside OnPreTick guarantees it). |
| 9 | `createContext(default, name) -> RUIContext` | handle = object identity + default; not slot-consuming | `TRuiContext<T>` typed handle (static/module-level) — typing removes Godot's Variant looseness; also holds the O(1) value stack (see gap table). |
| 10 | `useContext(key)` | no hook slot; records dep out-of-band; O(depth) walk (→ replace with cursor); default fallback | Typed read `UseContext(const TRuiContext<T>&) -> const T&`; dep recording for bailout re-check + propagation. |
| 11 | `provideContext(key, value)` | hook-style provider call (no wrapper element); change → eager subtree propagation | Keep family API; push/pop discipline through bailout/unwind paths is the correctness trap. |
| 12 | `useDeferredValue(value, deps?)` | returns previous value; commits target next frame via one-shot tick; optional deps gate | Back with `DeferredLane` instead of the ad-hoc tick; type-erased slot for the held value. |
| 13 | `useTransition() -> [is_pending=false, start]` | synchronous no-op, faithful to C# original | Trivial; keep signature parity. |
| 14 | `useStableCallback(cb)` | stable wrapper identity, always invokes the latest body; 0-arg | The pattern C++ NEEDS everywhere (TFunction-equality hole): wrapper `TFunction` reads slot's current inner. Promote to the default event-handler codegen strategy. |
| 15 | `useStableFunc(cb)` | alias of 14 | — |
| 16 | `useStableAction(cb)` | 1-arg variant | Templated arg type → per-instantiation slot type check; or erase arg via `FRuiValue`. |
| 17 | `useSafeArea() -> {left,top,right,bottom}` | `DisplayServer.get_display_safe_area()` px insets | `FDisplayMetrics::GetDisplayMetrics()` + `GetSafeZone`/TitleSafe margins (or `FSlateApplication::Get().GetSafeZoneSize`); note SSafeZone widget exists as declarative alternative. |
| 18 | `useSignal(sig, selector?, comparer?)` | subscribes to `RUISignal`, selector+comparer, re-bind every render, resubscribe on instance change, unsub on unmount; **subscribes during render** | Re-implement on **useSyncExternalStore semantics** (subscribe in effect, snapshot recheck) — fixes the restart-leak hazard; `TRuiSignal<T>` + `TFunction<S(const T&)>` selector = two type params. |
| 19 | `useSignalKey(key, initial, selector?, comparer?)` | process-wide registry (`RUISignals.get_or_create`) | FName-keyed type-erased registry; runtime type check on key collision. |

Cross-cutting C++ hook infrastructure: positional slot array + **three separate cursors** (state/effect/layout — copy Godot; keeps effect insertion from shifting state slots); thread_local `_cur` state pointer set by `_begin/_end` around render; hook-order validation (dev, kind-signature list, primed first render); set-during-render warn-once; slot teardown order on unmount = unsub external subscriptions → clear closures → drop slots (port `_dispose_fiber_state` verbatim — in C++ this is where `TFunction` capture cycles to `FRuiComponentState` are severed).

## 5. One-paragraph verdicts on the assignment's named questions

- **Double-buffering with a slab allocator — which is right?** Double-buffer. React reuses alternates; Godot converged to the same after measuring; C++ adds two more reasons (alloc churn, pointer stability for stored closures). Slab-allocate fiber pairs, free-list on deletion, no per-frame release.
- **Copy more from React — the ranked list:** (1) `childLanes`/dirty-bubbling subtree-skip bailout + `cloneChildFibers` — Godot's known missing piece, PORTING_PLAN 1.9; (2) React's two-pass child reconciliation with `lastPlacedIndex` minimal moves (replaces Godot's commit-time `_enforce_child_order`, which is wrong-shaped for Slate slot costs); (3) O(1) context cursor stack (replaces O(depth) walks); (4) React-19 ref lifecycle incl. cleanup callbacks (replaces call-every-commit); (5) `useSyncExternalStore` discipline for `useSignal` (subscribe-in-effect, fixes a real leak window); (6) StrictMode dev double-render; (7) React-19 `commitUpdate`-diffs-in-commit host config shape (no `prepareUpdate`) — which Godot already matches.
- **Keep from Godot (don't "upgrade" to React):** synchronous work loop + restart-from-root + atomic commit; 17-style effect list; direct-mutate hook slots (no update queue); eager context propagation; structural error boundaries (+ cooperative error latch, since UE disables exceptions); tiered list fast-paths (`_try_fast_leaf_list`, `_keys_stable`, `reuse_by_slot`), the apply-plan inline cache, and GO-05 node pooling — none exist in React and all matter more on Slate than on Godot.
