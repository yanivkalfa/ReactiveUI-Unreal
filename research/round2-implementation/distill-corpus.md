# ReactiveUI-Unreal — Master Research Distillation (12-agent round + adversarial critique)

## 1. ARCHITECTURE DECISIONS (D_coreArch)

### 1.1 Plugin & module layout — `ReactiveUI.uplugin`, four modules
| Module | Type | Deps | Contents |
|---|---|---|---|
| `ReactiveUICore` | Runtime | `Core` only — **no UObject, no CoreUObject** | VNode, fibers, reconciler, hooks, keys, callbacks, `IRUIHostConfig` (abstract host seam), signal store, context, router |
| `ReactiveUISlate` | Runtime | Core + `SlateCore`, `Slate`, `InputCore` | `FRUISlateHost : IRUIHostConfig`, element adapter registry, `SReactiveRoot`, widget pool, custom leaves (`SRUICanvas`) |
| `ReactiveUIUMG` | Runtime | above + `UMG`, `CoreUObject`, `Engine` | `UReactiveWidget : UWidget` embedding shim — the **only** UObject code |
| `ReactiveUIEditor` | Editor | above + editor modules | Nomad-tab mounting, debug/diagnostics UI |

Mirrors the Godot seam: reconciler touches the engine only through `IRUIHostConfig` (create/apply/insert/remove/reorder/pool-reset over opaque `FRUIHostHandle`), so Core compiles in a standalone `SlateViewer`-style program target that doubles as the CI test harness.

### 1.2 VNode & props — typed structs + set-bitmask, NOT prop bags
- **`FRUINode`** (transient value): `{ ETag Tag; FRUIElementTypeId Type; FRUIKey Key; TUniquePtr<FRUIPropsBase> Props; TArray<FRUINode> Children; void* ComponentId; }`. Heterogeneous children uniform because props erased behind `FRUIPropsBase`; only the element's adapter downcasts.
- Per-element props structs, e.g. `FRUIButtonProps : FRUIPropsBase { FText Label /*bit 0*/; bool bEnabled /*bit 1*/; FRUICallback OnClicked /*bit 2*/; uint64 SetBits; }` — `SetBits` records which fields the render actually specified.
- **Diff** = `Adapter->ApplyDiff(Widget, Old, New)`: downcast both, memberwise compare-and-set — flat, branch-predictable, zero allocation, no FName hashing.
- **Removed-prop contract via bitmask**: bit set last render but clear now → *ignored* for plain fields (family semantic "removed plain props don't reset"); event/ref/style bits *are* cleaned up.
- **Equality for bailout**: adapter `Equals(Old,New)` = set-bits + memberwise `==`, **excluding event fields** (`TFunction` has no `==`; events are always cheap trampoline-pointer updates and never force re-apply). `UseCallback` for identity-stable handles under memoization.
- Boilerplate macro/codegen'd: `RUI_PROP(FText, Label, 0)` generates field, builder setter, compare, apply row.
- **Rejected**: `TMap<FName, FRUIValue(TVariant)>` prop bag (Slate has no reflection — every key ends in a per-widget switch anyway; hashing, boxing FSlateBrush/FText, no compile-time checking). **Rejected**: `FInstancedStruct` (drags CoreUObject/USTRUCT into Core; reflected diffing slower than memberwise `==`). A generic `FName→FRUIValue` adapter can be added later behind the same adapter interface for the markup layer.

### 1.3 Component model
- **Free functions + explicit context handle, identity = function pointer**: `FRUINode Counter(FRUIContext& Ctx, const FCounterProps& Props);` used as `RUI::FC(&Counter, FCounterProps{...})`. `RUI::FC` templates over `(FnPtr, TProps)`, stores `ComponentId = (void*)FnPtr` (stable, comparable — solves bailout identity that `TFunction` alone cannot) plus a small invoker owning the typed props. Lambda components allowed, per-instantiation identity → "always re-renders" (documented, same as unmemoized inline React component).
- **Hooks via `FRUIContext&`** — thin wrapper over the current fiber's `FRUIComponentState*` passed into every component; replaces Godot's static `Hooks._cur`; no per-hook TLS lookup. Reconciler still sets a debug-only thread-local to assert single-threaded in-render usage. (⚠ CRITIQUE downgraded the "compile error" claim — see §6.)
- Hook storage ports 1:1: `FRUIComponentState` shared **by reference (`TSharedRef`) across fiber alternates**; positional `TArray<FRUIHookSlot>` (slot = `TVariant<FStateSlot, FMemoSlot, FRefSlot,…>`) with cursor reset per render; separate arrays/cursors for passive and layout effects; `UseContext` records deps out-of-band. Setters minted once per slot as `TSharedRef<FRUISetterCore>` closing over `(TWeakPtr<FRUIComponentState>, SlotIndex)`; `UseState<T>` returns copyable typed `TRUISetter<T>` handle → stable identity by pointer, safe after teardown via weak ptr (replaces Godot's index-bounds guard). Object.is-vs-deep-equality split kept (identity for containers in state, value equality for effect deps).
- **Rejected alternative**: CRTP class components (`TRUIComponent<FTodoList, FEmptyProps>` with `TRUIStateVar<T>` members) — hooks stop composing (custom hooks are plain functions taking `FRUIContext&`), reconciler would manage a second stateful object tree, diverges from both proven references, forfeiting the "twice-proven port" property.

### 1.4 Fiber memory model
- **Reconciler-owned slab allocator, raw pointers between fibers, zero ref counting inside the tree.** Godot's "sever cycles explicitly" generalizes to "don't create cycles at all". `FRUIFiber` in paged chunks (fixed addresses, free-list reuse); `Parent/Child/Sibling/Alternate` are raw `FRUIFiber*`; deletion = explicit `Release()` after teardown. Double-buffering 1:1 — matching positions reuse `Old->Alternate`, zero allocation on stable frames.
- Three ownership edges crossing the boundary: (1) `Fiber → TSharedPtr<SWidget>` strong (shadow tree keeps detached/pooled widgets alive — enables pooling); (2) `Fiber ↔ ComponentState`: state is `TSharedRef` on the fiber, its back-pointer to the live fiber is raw `FRUIFiber*` re-pointed each render/commit, nulled on teardown; (3) closures (setters, event trampolines) hold `TWeakPtr<FRUIComponentState>` / `TWeakPtr` proxies — never raw `this`, never strong cyclic refs. All Slate delegate binds use `CreateSP`.

### 1.5 Slate host config — adapters, setters-never-TAttributes, event proxies
- `IRUIElementAdapter` registry keyed by `FRUIElementTypeId`.
- **Create**: `SNew(SButton)` with *construct-only* args (styles, list-view columns) from props; then `ApplyFull`. Each adapter declares a **reconstruct-on-prop mask**: diffs touching those bits mark the fiber for widget replacement (same path as a type change).
- **Update**: constant values via setters (`SetText`, `SetVisibility`, `SetContent`, `SetImage`…). **Never bind delegate `TAttribute`s** — constants/setters keep widgets non-volatile and fast-path-valid under `Slate.EnableGlobalInvalidation`. Engine setters self-invalidate; host calls `Invalidate`/`MarkPrepassAsDirty` only on custom leaves (`SRUICanvas` = `draw_fn` analogue: paint callback stored in a proxy; `Invalidate(Paint)` only when callback identity or `RedrawKey` changes — register-once trampoline ports 1:1). (Critique verdict: CONFIRMED.)
- **Children**: `GetChildContainer()` per adapter (= `resolve_child_host`): panels expose slot ops (`AddSlot/InsertSlot(i)/RemoveSlot/NumSlots`); single-content widgets (`SBorder`, `SBox`, `SButton`) use `SetContent` and warn at >1 child (`warn_capacity`). Per-slot layout (padding, alignment, fill, ZOrder) is nested `FRUISlotProps` on the child vnode via UE 5.0 runtime slot setters.
- **Reorder**: no public MoveSlot → commit reorder pass **permutes children across existing slots via `DetachWidget`/`AttachWidget`** (no slot destruction), only on `structural` frames. (⚠ CRITIQUE: what this saves is slot construction/destruction, NOT ChildOrder invalidation — see §6.)
- **Events — bind once, swap callback pointer**: bind at construction to per-node `TSharedRef<FRUIEventProxy>` (`SetOnClicked(FOnClicked::CreateSP(Proxy, &FRUIEventProxy::HandleClick))`); proxy holds the current `FRUICallback`; diffs swap the pointer — no rebinding. Godot's runtime signal-name alias table becomes compile-time per-adapter event tables. Handlers return `FReply` (default `Handled()` for void callbacks — ⚠ critique gap on input swallowing).
- **Text**: raw `FText`/`FString` children auto-wrap to `STextBlock` vnodes; editable-text adapters skip `SetText` when equal (controlled-input caret preservation).

### 1.6 Keyed diff, bailout, reuse_by_slot, pooling — all port 1:1 algorithmically
- Three-tier keyed diff: fast-leaf-list in-place path → keys-stable positional scan → full keyed with persistent cleared-per-call `TMap<FRUIKey, FRUIFiber*>` + per-fiber `MatchedPass` mark-and-sweep counter; unkeyed children get sentinel positional keys.
- Bailout: `ComponentId` pointer equality ∧ no pending update ∧ context deps unchanged ∧ typed-props `Equals` (identity-first) ∧ children array identity; reuse cached `LastOutput`.
- `reuse_by_slot`: slot *i*'s widget reused for whatever leaf lands there (stateless childless leaves, stable type per slot) — churn becomes pure setter writes.
- `FRUIWidgetPool` keyed by element type id: unmounted poolable leaves `DetachWidget`-ed with last-applied props struct stashed; reuse = *diff apply* against stashed props (never eager reset-to-defaults — measured lesson). Cap 256/type, exclude stateful widgets (editable text, list views), drain on root unmount and in `ReleaseSlateResources`. Precedent: `SListView` row recycling / `FUserWidgetPool`.
- Scheduling: setter → `RequestUpdate()` → one-shot `FSlateApplication::OnPreTick` registration → single coalesced render+commit per frame before Slate prepass; mid-render setState restarts from root with the 25-restart guard.

### 1.7 Reconciler phases (1:1 from Godot/Unity)
Render: `begin_work` descending (reconcile children, run components with bailout) → `complete_work` ascending (create host node / flag `EFFECT_UPDATE`; build post-order effect list). Commit: deletions → placement/update/layout effects → reorder (structural frames only) → current↔wip swap → two-pass passive flush (all cleanups then all setups) → replay deferred updates. Structural error boundaries unchanged (UE builds without exceptions — same constraint as GDScript).

### 1.8 Mount surfaces
`SReactiveRoot : SCompoundWidget` (reconciler owns `ChildSlot`; root object holds reconciler, host, pool, pre-tick handle; `.Unmount()` runs cleanups, drains pool). Surfaces: (a) game viewport `GEngine->GameViewport->AddViewportWidgetContent(SNew(SReactiveRoot)…, ZOrder)`; (b) UMG `UReactiveWidget::RebuildWidget()` returns `SReactiveRoot`, `SynchronizeProperties` forwards designer props, `ReleaseSlateResources` unmounts; (c) editor `FGlobalTabmanager` nomad tab or `FSlateApplication::Get().AddWindow(SNew(SWindow)[…])` — Widget Reflector works natively (real `SNew` calls in adapters); (d) standalone program via `InitializeAsStandaloneApplication` — the automated test host for ported `core_test`/`update_test`.

### 1.9 Ports 1:1 vs rework
**1:1**: reconciler phases + effect list + commit order; double-buffered fibers; three-tier keyed diff + mark-and-sweep; bailout; hook slot/cursor model + state-shared-across-alternates; coalescing + restart guard + optional time-slicing; pool cheap-path; removed-plain-prop semantic (bitmask); structural error boundaries.
**Rework**: props (Dictionary → typed structs+bitmask+builders); prop application (reflective `node.set` → adapter setter tables + reconstruct masks); events (runtime signal discovery → compile-time delegate maps + `CreateSP` trampoline proxies); callable identity (Callable → `FRUICallback` shared-ref + `TWeakPtr` guards); memory (RefCounted+sever → slab + raw intra-tree pointers); scheduling (`call_deferred` → `OnPreTick`); reorder (`move_child` → slot detach/attach permutation); text (Label/String → STextBlock/FText); **style/classes needs the deepest rethink** — Slate styles are construct-heavy structs, not per-property channels; v1 ships per-widget style props + reconstruct path, stylesheet layer deferred (⚠ critique: styling is load-bearing — see §6).

### 1.10 D_coreArch OPEN QUESTIONS
Style/classes scope for v1; minimum engine version (UE 5.1+ assumed — `MarkPrepassAsDirty`, mature slot setters, `TSlateAttribute`); UMG UWidget-backed adapters inside the Slate tree vs pure-Slate; adapter/props codegen — macros vs offline generator; reorder benchmark spike (detach/attach vs remove+insert vs invalidation-boundary wrap, given UE 5.7 `FSlateInvalidationRoot` fast-path crash reports); threading stance (hard-assert game-thread vs structurally thread-agnostic render phase).

## 2. SLATE GROUND TRUTH (R_slate + R_ground)

### 2.1 Creation/mutation
- `SNew(WidgetType)` expands to `MakeTDecl<WidgetType>(#WidgetType, __FILE__, __LINE__, RequiredArgs::MakeRequiredArgs(__VA_ARGS__)) <<= WidgetType::FArguments()` — heap-allocates the widget as `TSharedRef`, builds stack `FArguments`, calls `Construct(const FArguments&)`. `SAssignNew` adds `.Expose(ExposeAs)`. File/line feeds Widget Reflector (Ctrl+Shift+W). Args declared via `SLATE_BEGIN_ARGS`/`SLATE_END_ARGS`: `SLATE_ARGUMENT` (construct-time only), `SLATE_ATTRIBUTE` (`TAttribute<T>`), `SLATE_EVENT`, `SLATE_SLOT_ARGUMENT`/`SLATE_NAMED_SLOT`.
- **Post-construction mutation is exactly what UMG itself does**: every `UWidget` caches a `TSharedPtr` to its SWidget, builds once in `RebuildWidget()`, pushes diffs via setters in `SynchronizeProperties` — the diff-based host is engine-precedented. Setters: base `SWidget` — `SetVisibility`, `SetEnabled`, `SetRenderTransform`, `SetRenderOpacity`, `SetToolTipText`, `SetCursor`, `ForceVolatile`, `Invalidate(EInvalidateWidgetReason)`, `MarkPrepassAsDirty` (UE 5.1+; `InvalidatePrepass` deprecated), `SlatePrepass`. Leaves: `STextBlock::SetText/SetFont/SetColorAndOpacity/SetJustification`, `SImage::SetImage`. Wrappers: `SBorder::SetContent`, `SBox::SetContent/SetWidthOverride`, `SButton::SetContent/SetButtonStyle/SetContentPadding/SetOnClicked`; `SCompoundWidget::ChildSlot.AttachWidget(...)`.
- Caveat: not every FArguments arg has a setter (style structs on complex widgets, `SListView` columns effectively construct-only) → per-widget "reconstruct on this prop" table required. Known wart: `STextBlock::SetTextStyle` historically didn't fully re-apply (hyaniner.com, 2024-05).

### 2.2 Dynamic children — TPanelChildren/FSlotBase
- Panels store children in `TPanelChildren<SlotType>` (slot objects owning child `TSharedRef` + layout params). `SVerticalBox/SHorizontalBox` (both `SBoxPanel`): `AddSlot()`, `InsertSlot(int32 Index)`, `RemoveSlot(const TSharedRef<SWidget>&)`, `ClearChildren()`, `NumSlots()`, `GetSlot(i)`. UE 5.0 slot refactor: `AddSlot` returns `FScopedWidgetSlotArguments`; runtime slot setters (`SetPadding`, alignment, size rules); `FSlotBase::AttachWidget(TSharedRef<SWidget>)` / `DetachWidget()` swap a slot's child in place (declarative `[ ]` = `operator[]` → `AttachWidget`).
- `SOverlay`: `AddSlot(ZOrder)`, `RemoveSlot`; draw order = add order unless per-slot ZOrder differs. `SGridPanel::AddSlot(Column, Row, Layer)`; `SCanvas`/`SConstraintCanvas` have `AddSlot`/`ClearChildren` with position/size slot attributes.
- **NO public MoveSlot on box panels** — reorder = remove+insert or detach/attach across slots. Cost: any child add/remove fires `ChildOrder` invalidation, the most expensive reason ("High"; "can cause significant performance bottlenecks if performed every frame") — rebuilds child list, re-lays-out subtree, and under global invalidation forces `FSlateInvalidationRoot::BuildFastPathWidgetList` rebuild.

### 2.3 Invalidation model
- `EInvalidateWidgetReason` bit flags: `Layout` 1<<0 (desired size changed, costliest routine reason), `Paint` 1<<1 (visual only, geometry cache valid), `Volatility` 1<<2, `ChildOrder` 1<<3 (implies Layout+Prepass — most expensive), `RenderTransform` 1<<4, `Visibility` 1<<5 (implies Layout), `Prepass` 1<<7 (re-measure recursively); combos `PaintAndVolatility`, `LayoutAndVolatility`.
- `SWidget::Invalidate(reason)` routes through `FastPathProxyHandle.MarkWidgetDirty()` to the owning `FSlateInvalidationRoot` (dirty list processed next frame).
- **Automatic**: engine widget setters already invalidate — UE5 `TSlateAttribute` members registered with a reason (`SLATE_ADD_MEMBER_ATTRIBUTE_DEFINITION(Initializer, Prop, EInvalidateWidgetReason::Paint)`); `TPanelChildren` add/remove invalidates `ChildOrder`; `SetVisibility` invalidates `Visibility`. **Manual**: only when mutating state read in `OnPaint`/`ComputeDesiredSize` outside official setters (custom leaves).
- Caching layers: `SInvalidationPanel`, Retainer Panel, **Global Invalidation** (`Slate.EnableGlobalInvalidation 1`, UE 4.25+, replaced LayoutCaching; each `SWindow` is an invalidation root). VALORANT shipped it on 4.25 (+4.26/4.27 backports): up to 15% client FPS, ~35% faster UI rendering — "single largest performance gain for clients since launch". Frequently-changing widgets should be `ForceVolatile(true)`. **UE 5.7 still has fast-path crash reports (`FSlateInvalidationRoot::ClearAllFastPathData` ensure) under heavy dynamic child churn** — the fragile spot for a reconciler.

### 2.4 TAttribute polling vs setters
Legacy `TAttribute<T>` bound to a delegate is polled every frame ("high memory overhead and not cache friendly" — UMG-Slate-Compendium); Epic's optimization guide reverses the old "polling over notification" philosophy at scale: prefer event-driven updates on the frame the value changes. UE 5.0 `TSlateAttribute`/`TSlateManagedAttribute` update once per frame in prepass and register an invalidation reason. **For a diff-based host: pass constant TAttributes (a constant attribute stores a value, no delegate call) and write via setters on change — optimal Slate citizenship**, same model as UMG.

### 2.5 Memory/lifecycle/threading
- Pure `TSharedRef/TSharedPtr/TWeakPtr`; no GC, no UObject. Parents own children strongly through slots; child back-pointer `SWidget::ParentWidgetPtr` is `TWeakPtr<SWidget>` (`AssignParentWidget`/`GetParentWidget`/`ConditionallyDetatchParentWidget`) — no cycles; destroying a window cascades.
- Dangling risks: `CreateRaw`/`CreateLambda` capturing raw `this` → bind with `CreateSP` (weak via `TSharedFromThis`); UMG↔Slate leaks via `FUserWidgetPool` not released in `ReleaseSlateResources`.
- Reconciler holding a `TSharedPtr` shadow tree keeps detached widgets alive for free — enables pooling. In-engine pooling precedent: `SListView/STreeView/STileView` create rows only for visible items via `OnGenerateRow` and recycle (100+ items → handful of live rows); UMG `UListView` uses `FUserWidgetPool` (`GetOrCreateInstance`/`Release`/`ResetPool`).
- Threading: FieldNotify delegates game-thread-only (`FNotThreadSafeNotCheckedDelegateUserPolicy`); Slate is game-thread for layout/paint-element generation (per-frame order Prepass → Tick → Paint via `FSlateApplication::Tick → TickAndDrawWidgets → SWidget::Paint → OnPaint`); no explicit render-thread split is exposed to a host — the critique flags that the designs never wrote the threading contract down (see §6).

### 2.6 Layout/paint/scale/events
- Two-pass layout: prepass/`CacheDesiredSize` bottom-up (`ComputeDesiredSize` on leaves), `ArrangeChildren` top-down; `OnPaint(FPaintArgs, FGeometry, FSlateRect, FSlateWindowElementList&, int32 LayerId, …)` emits `FSlateDrawElement::MakeBox/MakeText/MakeLines` with integer LayerId z-order. Profile: `stat slate` ("Num Painted Widgets").
- Scale: entire Unreal Editor is Slate; Epic's UMG guide: complex screens have "well over 1000 children" with hundreds visible; off-screen Hidden (vs Collapsed) widgets still cost layout. Thousands of live widgets fine; thousands *painted per frame without invalidation caching* is where it bites.
- Events: `SLATE_EVENT` at construction (`.OnClicked(this, &S::Handler)`, `_Lambda`, `_Raw`, `_Static`); handlers return `FReply::Handled()/Unhandled()` (+`.CaptureMouse()` etc.). **Rebinding after construction works**: `SButton::SetOnClicked(FOnClicked::CreateLambda(...))` documented practice; `SBorder` exposes `SetOnMouseButtonDown/Up/Move/DoubleClick`; editable text has change/commit delegate setters. Where no setter exists → bind-once trampoline.
- Standalone host: `FSlateApplication::InitializeAsStandaloneApplication(GetStandardStandaloneRenderer())` → `AddWindow(SNew(SWindow)…)` → loop `PumpMessages(); Tick(); FTicker::GetCoreTicker().Tick(dt);` at ~60fps → `FSlateApplication::Shutdown()` (SlateViewer, UnrealPakViewer). `FSlateApplication` also offers `FindPathToWidget`, `SetAllUserFocus`, `PushMenu`.

### 2.7 Godot/Unity ground truth to preserve (R_ground)
- Host contract is exactly two files in Godot (`host_config.gd` RUIHost + `style.gd` RUIStyle): `create_node`, `resolve_child_host`, `apply_props` (6 ordered steps: event disconnect → prop set → ref every commit → style/classes merge+diff → items registry → draw_fn trampoline), `reset_for_pool`/`reset_removed_plain`, `warn_capacity`, plus `add_child/move_child/queue_free` leaks. RESERVED props: `key, ref, style, classes, children, items, draw_fn, redraw_key, reuse_by_slot`.
- Unity `FiberHostConfig.cs` + `ElementRegistry` + `ITypedElementAdapter.ApplyTypedDiff/ApplyTypedFull` over `BaseProps` structs is **the cleaner seam to copy** for C++.
- Fiber fields: `parent/child/sibling/index`, `tag/key/type/component`, `props/pending_props/input_children`, host `node`, `portal_target`, error-boundary fields, `alternate`, `effect_tag`+`next_effect`, `deletions`, `matched_pass`, `provided_context/reads_context`, `has_pending_update/subtree_has_updates`, shared `state`, cached apply plan (`apply_size/apply_special/apply_plain`).
- The `items` seam (declarative item-model adapters with identity-based "tracker" state preservation, multiset-counted duplicate ids) is the Godot precedent for the missing SListView adapter.
- Perf lessons to inherit day one: double-buffered fibers (zero alloc stable frames); fast-leaf-list + per-row bailout (biggest win for big dynamic lists); `reuse_by_slot`; pool cheap-path (eager reset-to-defaults cost ≈ instantiate; instead detach + stash + diff-apply; cap 256/class); per-fiber apply-plan inline cache; identity-first comparisons; allocation hygiene (persistent `_key_map`, `matched_pass` flag not per-frame dict, GO-06 eliminated 12 empty-dict allocs per styled node per frame, reorder only on structural frames, `has_meta` gates); coalesce one render/frame + 25-restart guard.
- Semantics to preserve: removed plain props NOT reset (events/style/refs/draw ARE); Object.is for state, deep equality for effect deps; controlled-text caret preservation; String children auto-wrap.

## 3. INTEROP (D_interop + R_mvvm)

### 3.1 FieldNotify ground truth (engine-level, plugin-free) — critique verdict: CONFIRMED
- `INotifyFieldValueChanged` lives in engine runtime module **`FieldNotification`** (`/Engine/Source/Runtime/FieldNotification/Public/INotifyFieldValueChanged.h`). API: `FDelegateHandle AddFieldValueChangedDelegate(UE::FieldNotification::FFieldId, FFieldValueChangedDelegate)`; `bool RemoveFieldValueChangedDelegate(FFieldId, FDelegateHandle)`; `int32 RemoveAllFieldValueChangedDelegates(FDelegateUserObjectConst)`; `void BroadcastFieldValueChanged(FFieldId)`; `const UE::FieldNotification::IClassDescriptor& GetFieldNotificationDescriptor() const`.
- `FFieldValueChangedDelegate` = `TDelegate<void(UObject*, UE::FieldNotification::FFieldId)>` with `FNotThreadSafeNotCheckedDelegateUserPolicy` → **game-thread only**. `FFieldId`: `GetName()`, `GetIndex()`, `IsValid()`.
- UHT generates per-class `FFieldNotificationClassDescriptor : Super::FFieldNotificationClassDescriptor` with `static const FFieldId <Name>` per FieldNotify member, `enum EField { IndexOf_<Name> = SuperDescriptor::Max_IndexOf_ + 0, Max_IndexOf_ }` chained across hierarchy, and `ForEachField(const UClass*, callback)` (how `UE::MVVM::Private::GetAvailableBindings` discovers fields). Canonical subscribe uses `CreateUObject` (binds weakly).
- FieldNotify functions: `UFUNCTION(FieldNotify, BlueprintPure)`, `const`, 0 args, single return. `UE_MVVM_SET_PROPERTY_VALUE(Member, NewValue)` (compare+assign+broadcast, returns bool) and `UE_MVVM_BROADCAST_FIELD_VALUE_CHANGED(Member)` are **plugin-side**; C++ dependency cascades are manual and cycle-unguarded; Blueprint "Set w/ Broadcast" cascades automatically.
- `UFieldNotificationLibrary` (`BroadcastFieldValueChanged`, `BroadcastFieldsValueChanged`, `SetPropertyValueAndBroadcast`) is Engine module. Stock UMG widgets (`UEditableText`, `USlider`, `UImage`) implement the interface.
- `UMVVMGameSubsystem` (UGameInstanceSubsystem) → `GetViewModelCollection()` → `UMVVMViewModelCollectionObject` (`AddViewModelInstance(FMVVMViewModelContext{ContextClass, ContextName}, UMVVMViewModelBase*)`, `FindViewModelInstance`, `FindFirstViewModelInstanceOfType`, `RemoveViewModel`, `RemoveAllViewModelInstance`, `OnCollectionChanged`); **collection typed to `UMVVMViewModelBase*`** — interface-only stores can be set on views (`UMVVMView::SetViewModel(FName, TScriptInterface<INotifyFieldValueChanged>)`) but not registered globally.
- Compiled bindings are event-driven: `UMVVMViewClass` + `FMVVMCompiledBindingLibrary` (`GetFieldId` returns `TValueOrError<FFieldId, void>`); per-instance `UMVVMView` (a `UUserWidgetExtension`) does `InitializeSources()` → `InitializeBindings()`. `EMVVMExecutionMode`: Immediate, **Delayed** (flushed once end-of-frame before drawing), Tick (only polling mode), DelayedWhenSharedElseImmediate (default "Auto"). Community benchmark: 10,000 changes/frame = 976.7 ms Immediate vs 12.8 ms Delayed (tsubasamusu.com) — ⚠ do NOT borrow as "76x faster" claim, see §6.
- Status: Viewmodel Beta in 5.7 AND 5.8; Cody Albert (May 21 2025): used heavily internally, "no significant feature work left", out of beta "soon", "No breaking changes planned". Native in UEFN/Fortnite; 5.7 MetaSound Viewmodels; 5.8 MetaSound tabs "built on Unreal's MVVM framework". Alternatives: MDViewModel (DoubleDeez, MIT, UE 5.1+, 56★, v1.0 Sep 8 2024 — binds Actor/Object BPs, View Model Providers Cached/Unique/Manual, BP-debugger integration); druhasu/UnrealMvvm (own `TBaseViewModel`, bypasses FieldNotify).

### 3.2 UseField / UseViewModel design
```cpp
float HP = RUI::UseField<float>(VM, "CurrentHealth");        // field-granular, preferred
auto Reader = RUI::UseViewModel(VM);                          // whole-VM read-tracking
float HP  = Reader.Get<float>("CurrentHealth");
FText Tag = Reader.Get<FText>("GetHealthLabel");              // FieldNotify function
```
- Resolve `FFieldId` once via `VM->GetFieldNotificationDescriptor().GetField(VM->GetClass(), FName(Name))`; memoize per `(UClass*, FName)` alongside resolved `FProperty*` / `UFunction*` (const, 0-arg, single return; read via `ProcessEvent` on cached param buffer).
- Subscribe on mount commit (passive-effect phase, game thread): `Handle = VM->AddFieldValueChangedDelegate(FieldId, FFieldValueChangedDelegate::CreateSP(HookState, &FVMSub::OnChanged));` — `CreateSP` against the `TSharedRef` hook-state self-invalidates if fiber dies first; cleanup still calls `RemoveFieldValueChangedDelegate` explicitly so the multicast list compacts.
- On broadcast: re-read, compare via `FProperty::Identical`, only if changed → mark fiber dirty → `RequestUpdate()` → coalesced once/frame (Delayed-mode-like batching; defensible claim is "we batch like Delayed", NOT 76x).
- Read tracking (`UseViewModel`): Reader records each FieldId touched during render; post-commit diff vs previous set — subscribe new, remove stale.
- Lifetime: hook cells hold VM as **`TWeakObjectPtr<UObject>`** by default (observe, never own); stale → prop-supplied default + optional `EStaleVMPolicy::RenderDefault | Unmount`. `UseOwnedViewModel<T>()` for UI-local VMs: `NewObject<T>()` in **`TStrongObjectPtr`** (root set), released in unmount cleanup. Unsubscribe guards `VMWeak.IsValid()`.
- Reverse direction: `URuiSignalViewModel` implements `INotifyFieldValueChanged` (gist pattern: `FFieldMulticastDelegate` + `TBitArray` enabled-bits + hand-rolled descriptor) so **UMG MVVM views bind to ReactiveUI state** via `UMVVMView::SetViewModel`. Separate plugin-gated module `URuiViewModelBridge : UMVVMViewModelBase` solely for Global Viewmodel Collection registration. Core stays plugin-free.

### 3.3 UMG embedding, both directions
- **(a) ReactiveUI inside a UserWidget**: `UReactiveHostWidget : UWidget` (D_interop naming; D_coreArch calls it `UReactiveWidget` — reconcile in plan), designer-placeable; `RebuildWidget()` → `FReactiveRoot::Create(...)->GetSlateWidget()`; `ReleaseSlateResources` → `Root->Unmount()`. Blueprint passes initial props + VM (flows via context → `UseField`).
- **(b) UMG widgets as host elements**: `V::Umg(UMyWidget::StaticClass(), Props)` — `CreateWidget<UUserWidget>(OwningPlayer, Class)` (owning player threaded down from root — required for CommonInput/localization), `TakeWidget()` yields `SObjectWidget`. Fiber host-node handle holds the UUserWidget in `TStrongObjectPtr`, released in deletion commit after removing the Slate child. Props→widget: reflection via `FProperty`, prefer `Set<Name>` UFUNCTION; FieldNotify targets via `UFieldNotificationLibrary::SetPropertyValueAndBroadcast` (widget's own bindings still fire). Events: React-style props bound to dynamic multicast delegates via pooled UObject trampoline (delegates require UObject targets); unbound on cleanup. Cache per-class prop maps.

### 3.4 CommonUI citizenship — two rules, six delegations
1. **Screens live *inside* activatables, never replace them**: `URuiActivatableScreen : UCommonActivatableWidget` embedding a `UReactiveHostWidget`; pushed onto existing `UCommonActivatableWidgetContainerBase` stacks; activation/input-mode/back-handling/focus restoration stay CommonUI's job. `UseActivation()` hook surfaces `OnActivated/OnDeactivated`; `autofocus` prop → `GetDesiredFocusTarget()`.
2. **Common widgets wrap fine as children** (`V::Umg(UCommonButtonBase...)`) via owning-player plumbing; wrapping `UCommonActivatableWidget` subclasses as plain children is **rejected with a diagnostic** pointing at `URuiActivatableScreen`.
Never rebuild: activatable stacks/back-stack; `UCommonUIActionRouter` input routing (never install own input preprocessor); `UCommonInputSubsystem` input-method detection + platform glyphs (`UseInputMethod()` reads + rebroadcasts); gamepad/keyboard focus navigation & analog cursor (standard Slate nav over real SWidgets, zero custom focus manager); `UCommonButtonBase` states/held-input/text styles (style props map onto Common style assets); platform traits/safe zones (`UseSafeArea` reads `SSafeZone`/trait tags).

### 3.5 Migration story (shippable at every step)
1. Keep VMs unmodified (`UseField`). 2. Leaf islands: one `UReactiveHostWidget` in an existing UserWidget sharing the same VM. 3. Screen-level `URuiActivatableScreen`s on existing CommonUI stacks; legacy Common widgets via `V::Umg`. 4. Optional inversion: ReactiveUI root, designer pieces as `V::Umg` islands. Rollback = delete a widget.

### 3.6 D_interop OPEN QUESTIONS
Markup compile target (C++ vs interpreted asset vs both); read-tracking reflection cost per frame (`FProperty::ContainerPtrToValuePtr` vs compile-time descriptor fast path `UseField<&Descriptor::Field>`); FieldNotify cascade cycle-breaker (per-frame FieldId de-dup?) or leave Epic's semantics; UMG designer preview vs placeholder box for v1; TStrongObjectPtr vs SObjectWidget's existing GC keep-alive double-hold; paid Fab tier containing MVVM bridge vs strictly tooling-only; CommonUI integration as optional module (zero CommonUI includes in core — SlateIM signals Epic rethinking); **naming collision with .NET ReactiveUI** in the Unreal/C#-adjacent space.

## 4. DX PLAN (D_dx + R_dx)

### 4.1 Iteration-loop options, ranked
1. **(c) Runtime-interpreted markup** — <1 s, state-preserving, mid-PIE and on device; can't express arbitrary C++ → **primary dev loop**.
2. **(b) Markup → codegen'd C++ (PreBuildSteps)** — 5–20 s via Live Coding *if* changes stay in `.cpp` bodies; minutes on header churn → **ship path, not dev loop**.
3. **(a) Pure C++ + Live Coding** — 3–15 s for `.cpp` bodies; any UPROPERTY/UFUNCTION/signature/header change → restart in practice; `.cpp` constructor defaults never propagate; Object Reinstancing needs manual pointer cleanup (`ReloadReinstancingCompleteDelegate`/`ReloadCompleteDelegate`) or shutdown crashes → baseline for logic only. Competing DX bar is Blueprint's 1–3 s + WYSIWYG.
4. **(e) UnrealSharp** — seconds-scale C# reload, vanilla engine, MIT, .NET 10 (10.0.5+), UE 5.6–5.8, 1,872★, no tagged releases, Win/Mac only, jam-scale → optional Phase 3+ adapter, never a foundation.
5. **(d) AngelScript (Hazelight fork)** — best-in-class (save→instant, mid-PIE non-structural), shipped It Takes Two + Split Fiction (1.7M+ lines / 16,000+ files), Embark (THE FINALS, ARC Raiders) — but engine fork incompatible with pre-built binary plugins → kills Fab distribution; community-maintained adapter only.
Verse: UEFN-only until UE6 EA (end 2027); "not intended for general adoption at this point".

**Phased plan — "interpret in dev, compile to ship"**: Phase 1 C++ core (fiber reconciler + hooks + `FRuiSlateHost`) with `V.*`-style factories; Phase 2 `.uetkx` interpreted at runtime (the DX headline — file-watch → reparse → diff → sub-second state-preserving patch, the loop Noesis/Gameface proved commercially); Phase 3 codegen backend (same `.uetkx` → plain C++ `Render()` for cooked builds; CI parity test that both produce identical vtrees); Phase 4 on-device remote reload (editor pushes specs over TCP); optional UnrealSharp/AngelScript adapters.

### 4.2 `.uetkx` markup
- Abstract element names (`Border/VBox/HBox/Text/Button/Image/ScrollBox/List…`) mapped by host config to `SBorder/SVerticalBox/…`; escape hatch `<slate:SConstraintCanvas>`. ⚠ Extension inconsistency across docs: `.uitkx` (Design 3/interop) vs `.uetkx` (Design 2/dx) — must be reconciled; family byte-compatible grammar with `.guitkx` is the ship gate.
- **Two-backend constraint shapes the embedded language**: raw C++ can't be interpreted, so markup expressions are a small **bindable expression DSL** (literals, prop/state paths, arithmetic/comparison, indexing, `fmt()`), and **imperative logic lives in a C++ logic class referenced by name** (`@class_name ShopPanel` / `@logic UShopLogic`), mirroring the repo's `.hooks.guitkx` split and benui's `BindWidget` convention. Directives: `state selected: int32 = -1`, `@for item in items key={item.Id}`, `@if selected >= 0`, `onClick={ OnBuy(item.Id) }`, built-in `set(count, count + 1)` for markup-only local state.
- **Codegen output (Phase 3)**: deliberately **plain C++, zero UHT surface** — no UCLASS/UPROPERTY in generated files, so regeneration never triggers reflection/reinstancing and Live Coding stays on its safe path. `ShopPanel.uetkx.g.h` = shape-stable (`struct FShopPanelProps`; `struct FShopPanel : FRuiComponent<FShopPanelProps> { FRuiNode Render(FRuiCtx&) override; }`), regenerated only on prop-signature change; `.g.cpp` holds the body where all expression edits land (Live Coding-friendly). Handlers compile to lambdas capturing `this` calling the logic class directly (type-checked).
- **UBT integration**: `PreBuildSteps` in `.uplugin` invokes the compiler (`$(ProjectDir)`/`$(PluginDir)`/`$(EngineDir)`/`$(TargetName)`/`$(TargetPlatform)`/`$(TargetConfiguration)` expansion; UBT writes .bat/.sh into `Intermediate/Build/...`; precedent unreal.hx, protoc plugins). **⚠ THE PITFALL (critique-confirmed)**: UBT's cached makefiles gather the source-file list BEFORE pre-build steps execute — adding/removing source files invalidates the makefile and requires re-gathering/project regeneration, so "create a new component" generates a `.g.cpp` the same build won't compile. Precedents only regenerate known file sets. This is NOT solved; the plan must add a mitigation (e.g., pre-registered unity/aggregate .cpp that includes generated bodies, or explicit project-regen step, or compile-into-known-file-set design).
- **UHT constraint**: UHT plugins (`*.ubtplugin.csproj`, `[UnrealHeaderTool]` class, `[UhtExporter(Name=…, ModuleName=…, Options=…)]`, referencing `EpicGames.UHT`, documented UE 5.3) are **exporters only** — no script generation. Use a UhtExporter to **dump reflection JSON** (available UFUNCTIONs, struct fields) into Intermediate; the markup compiler + LSP consume it to typecheck `@logic` handler names and prop paths offline. Existing TypeScript LSP/TextMate/VS Code+VS2022 stack reused with a `uetkx` schema.
- **clangd/virtual-document design**: embedded-expression intelligence follows the Godot family pattern — LSP answers markup intelligence from schema locally; embedded-code intelligence via a synthetic virtual document with a length-preserving source map (the family's `@gdscript-analyzer/core` precedent), here typechecked against the UHT reflection JSON dump rather than a live engine.

### 4.3 Runtime-interpreted path
- Parser (C++, shared lexer grammar with the compiler) → `FRuiNodeSpec { FName Element; TMap<FName, FRuiExpr> Attrs; FRuiExpr Key, If; TOptional<FRuiForSpec> For; TArray<FRuiNodeSpec> Children; TArray<FRuiHandlerRef> Handlers; }`; `FRuiExpr` = parsed DSL AST (literal | path | binop | fmt | index).
- In-editor it lives in `URuiMarkupAsset : UPrimaryDataAsset` (retains source text; binary-serialized spec for cook — cooked builds *can* interpret for device hot-tuning, codegen is default ship). ⚠ Critique: this contradicts "text files you can diff"; import pipeline (UFactory/reimport, cook rules, redirects) unspecified — must be resolved (loose-file source-of-truth + import).
- Each render the interpreter walks the spec evaluating `FRuiExpr` against context (props + hook state + logic object via UE reflection) and emits the same `FRuiNode` vtree as codegen — one reconciler downstream.
- **Handlers**: logic class is a normal `UCLASS() UShopLogic : URuiLogic` with `UFUNCTION() void OnBuy(int32 ItemId);` — interpreted mode resolves by `FindFunction(FName)` + `ProcessEvent` with DSL-evaluated args (typed against reflection dump; mismatches = LSP diagnostics AND runtime warnings). Non-UObject users: `RUI_HANDLER(Name, Lambda)` manual registry macro. Codegen mode binds same names as direct calls.
- **What is interpreted vs compiled**: interpretation covers structure, styling, simple conditions (`@if`), loops (`@for` only), wiring, and the curated pure-function whitelist (`fmt`/`clamp`/`lerp`); NOT arbitrary statements, closures, match-dispatch, arbitrary function calls, or new types. Rule of thumb: "structure, styling, simple conditions, and wiring live in markup; anything else is a one-line handler in C++" — which Live Coding patches in seconds as a `.cpp` body.
- **HMR loop mechanics**: `IDirectoryWatcher` (DirectoryWatcher module) on content roots for `*.uetkx`; on change → reparse → validate (parse errors → Message Log with de-duplicated file:line links; **last-good spec stays live — never crash the session**) → swap spec on asset → `URuiSubsystem` marks mounted roots dirty → next-frame re-render. PIE: watcher runs in editor process, identical path mid-play. Standalone dev builds: watcher module ships in non-editor dev targets + `rui.reload` console command. Device (Phase 4): serialized specs over a socket.
- **State-preservation rule (critique-corrected)**: "state survives" holds ONLY for structure/style/expression edits that don't touch the hook signature; adding/removing/reordering a `state` declaration (or a conditional branch) shifts positional hook slots → state scrambled or type-mismatched. The plan MUST define hook-signature hashing: hash the ordered (slot-type, declared-name, type) signature per component; on mismatch, migrate by declared state name where possible, else documented reset — no silent scrambling.

### 4.4 Debugging / devtools / testing
- **ReactiveUI Inspector** (editor `SDockTab`): live fiber tree; per-fiber props, hook slots (index/type/value), render count, last-render µs; flash-on-rerender (bailout-failure visibility); click-through to Epic's Widget Reflector selection.
- Runtime: `rui.stats` (renders/frame, fibers, patch counts — the Doom demo perf-overlay pattern), `rui.DumpTree`; optional ImGui overlay module (soft dep on IDI-Systems fork) for shipping-build inspection.
- **Testing, three layers**: (1) Core unit — `FRuiMockHost`, Automation Specs (`BEGIN_DEFINE_SPEC`), no Slate/RHI (port of `core_test.gd`/`update_test.gd`); (2) Slate integration — real SWidget trees under `FSlateApplication` with `-nullrhi`; (3) Markup + parity — golden-file spec snapshots + generated C++ compiled in CI + **parity suite rendering every example both ways asserting identical vtrees** (analogue of `demos_test.gd`). CI: `UnrealEditor-Cmd.exe <Project> -ExecCmds="Automation RunTests ReactiveUI; Quit" -nullrhi -unattended -nosplash -log`, wrapped in UAT for multi-engine-version cook-and-test.

## 6. THE CRITIQUE — every gap + resolution, and the downgraded claims

### 6.1 Gaps (21) — each with required resolution/mitigation
1. **Localization unsolved** (.uetkx strings never enter FText gather; interpreted mode culture-invariant; API sketch uses INVTEXT; no live-culture re-render) → define a `loc("Namespace","Key","Text")` markup form compiled to LOCTEXT/string-table references, register `.uetkx` with the localization gather pipeline, and subscribe roots to culture-change events to force re-render.
2. **Brush/asset-reference story missing** (FSlateBrush textures/materials: soft paths? sync/async? GC of UTexture/UMaterial held from non-UObject fibers) → define an `FRUIAssetRef` (soft object path + async-load policy) and a single `FGCObject`/`AddReferencedObjects` root on the mount surface owning all brush-referenced UObjects.
3. **Animation entirely absent** (no FCurveSequence analogue, no transitions; exit animations touch the reconciler via delayed unmount) → schedule a v1 animation primitive (`UseAnimate`/interpolated props on the OnPreTick pass + a delayed-unmount/exit-transition protocol) — retrofit cost is reconciler-level, so design it now even if shipped later.
4. **No list virtualization** (5,000-row inventory = 5,000 SWidgets — exactly the demand-cited hitching) → build the SListView adapter mapping declarative children onto `OnGenerateRow`/recycling (the Godot `items` seam is the template); this is a named hard deliverable, not an exclusion.
5. **No portal mechanism** (tooltips/menus/dropdowns/drag previews/modal SWindows live via `FSlateApplication::PushMenu/AddWindow`) → design a React-portal analogue with explicit ownership: portal fibers own the out-of-tree content and tear it down on unmount.
6. **Focus lifecycle across diffs undefined** (reconstruct-mask replacement or keyed reorder can destroy/move the focused widget mid-commit; FSlateUser/split-screen; navigation config under sibling permutation) → capture focused-widget path pre-commit per FSlateUser and restore post-commit; document reconstruct = focus loss unless keyed-restored.
7. **Commit-phase reentrancy has no contract** (Slate setters fire delegates synchronously — SetText→OnTextChanged — so user handlers can call hook setters during commit: neither the 25-guard case nor the pre-tick case) → define: setter calls during commit are queued to the deferred-update replay (already in commit order) and assert-documented; generalize the editable-text skip-when-equal to all self-notifying setters.
8. **No threading contract** (FieldNotify is game-thread-only; nothing says whether setters/RequestUpdate may be called from async/network callbacks) → policy: `checkf(IsInGameThread())` in all public entry points + a documented `AsyncTask(ENamedThreads::GameThread)` marshaling helper (`RUI::PostToGameThread`).
9. **Per-render allocation budget unexamined** (TUniquePtr<FRUIPropsBase> per vnode + TArray children + TFunction captures ×thousands ×60fps; slab covers fibers only) → add a per-frame vnode arena (frame allocator reset after commit), a `STATGROUP_ReactiveUI` stat group, port the Godot bench suite, and set target numbers vs UMG/MVVM baselines before perf claims ship.
10. **World-space UI unaddressed** (UWidgetComponent/render-target, hit testing, retainer-box interactions) → verify and document `UReactiveWidget`-in-a-`UWidgetComponent` including retainer/render-target invalidation behavior; add an example.
11. **UMG Designer design-time behavior** (`RebuildWidget` runs hooks/effects/VM subscriptions in the Designer preview world) → explicit design-time stub: render a placeholder (or a declared static preview) when `IsDesignTime()`, never running user effects at edit time.
12. **Blueprint-only authoring gap** (handlers need a C++ UFUNCTION class or registration macro; Fab's largest demographic is BP-heavy) → state explicitly that `@logic` may be a Blueprint class (FindFunction resolves BP functions — verify + test), and size the BP-only market segment honestly in positioning.
13. **Markup source-of-truth contradiction** (`URuiMarkupAsset : UPrimaryDataAsset` retaining source vs loose-file watcher vs "text files you can diff"; extension `.uitkx` vs `.uetkx`; import/reimport/cook/redirect rules unspecified) → resolve: loose `.uetkx` text files are the source of truth (diffable pitch preserved); a UFactory imports to the asset which holds only the compiled spec; cook ships spec and/or generated C++ per an explicit rule; standardize on `.uetkx`.
14. **Codegen file ownership unspecified** (where .g.h/.g.cpp live, who wires Build.cs/includes, stale-file cleanup, compiler-binary distribution per build-host OS/CI) → decide: generated files land in the game module under `Intermediate`-adjacent known paths with a generated aggregate registered up front; compiler ships as C++ in-plugin built for Win/Mac/Linux; document Build.cs wiring + clean rules.
15. **Input consumption policy** (default `FReply::Handled()` swallows events, breaks bubbling/drag/touch; no drag-and-drop API, no shortcut story, no analog/gamepad policy outside CommonUI) → make handled-vs-unhandled explicit per event prop (typed return or `bHandled` flag), add drag-drop and key-shortcut APIs to the roadmap, document gamepad delegation to CommonUI.
16. **Interpreted-mode runtime errors** (DSL evaluation error mid-render — bad path, `items[selected]` out of range — has no defined behavior; structural boundaries can't catch child crashes) → define: expression evaluation is total — errors yield a typed default + a Message Log diagnostic + a visible dev-build error leaf widget; never crash the render.
17. **Lifetime vs world teardown** (URuiSubsystem type unchosen; roots/pool hold TSharedPtr<SWidget>/TStrongObjectPtr across level travel and PIE end; ReleaseSlateResources vs subsystem deinit vs GC ordering) → choose subsystem type explicitly (likely UGameInstanceSubsystem with per-world root tracking), define a teardown order contract (unmount roots + drain pool on world teardown BEFORE GC), and test PIE end/level travel in CI.
18. **Accessibility never mentioned** (FSlateAccessibleWidget, platform cert) despite "accessibility already understands our widgets" in positioning → verify the claim: emitting real SWidgets should inherit Slate's accessibility layer — test with the screen-reader path and either substantiate or soften the copy.
19. **Styling deferred but load-bearing** (every markup example uses style dicts Design 1 defers; style bits on the reconstruct mask means any style tweak = widget replacement — focus loss + ChildOrder cost, undermining the hot-reload demo whose primary use is style tweaking) → v1 must ship at least a minimal style-dict→setter mapping for the hot-path properties (colors, font size, padding) that does NOT sit on the reconstruct mask, reserving reconstruction for genuinely construct-only style structs.
20. **Team-size vs scope math absent** (4 modules + 2-backend toolchain + LSP + inspector + MVVM bridges + CommonUI + 3-version CI = multiple engineer-years; support is "a full time job"; mean Fab earnings ~$1.2K/yr; UE6 lands mid-support-window) → the plan needs an explicit v1 cut-line and phase-gated resourcing; treat Phases 3–4 as demand-gated.
21. **Hot-reload state-migration rule missing** (add/remove/reorder of a `state` declaration shifts positional slots → scrambled/type-mismatched state) → hook-signature hash per component (ordered slot types + declared names); on hash change, migrate by declared name where types match, reset otherwise, and say so in the docs — never silently scramble.

### 6.2 Challenged claims — corrections that MUST survive into the plan
- **FieldNotify zero-plugin-dependency — CONFIRMED.** `INotifyFieldValueChanged`/`FFieldId`/delegates/UHT descriptors and `UFieldNotificationLibrary` are engine-level; only authoring macros (`UE_MVVM_SET_PROPERTY_VALUE`) and `UMVVMViewModelBase`/global collection are plugin-side — the gated bridge module handles that correctly. Consume, don't author.
- **Reorder via DetachWidget/AttachWidget "minimizes ChildOrder invalidations" — DOUBTFUL (misattributed).** Re-parenting marks the parent ChildOrder-dirty exactly like remove+insert; both paths rebuild the flattened fast-path widget list (the most expensive invalidation, with live UE 5.7 `FSlateInvalidationRoot::ClearAllFastPathData` crash reports under churn). What permutation actually saves is **slot-object construction/destruction**, not ChildOrder invalidation. Reorder-only-on-structural-frames stays right; the headline benefit must be restated and **benchmarked** (spike: detach/attach vs remove+insert vs invalidation-boundary wrapping).
- **Setters-never-delegate-TAttributes = ideal Slate citizen — CONFIRMED.** Matches Epic's at-scale guidance, TSlateAttribute design intent, and Global Invalidation (VALORANT's 15%). Unproven corner: whether per-frame prop diffing itself stays cheap at scale — covered by the perf-measurement plan (gap 9).
- **<1 s state-preserving HMR — PLAUSIBLE, guarantee overstated.** Mechanics sound (IDirectoryWatcher, spec swap, last-good spec; Noesis proves the loop commercially) — but Noesis preserves state because state lives in the ViewModel; here it lives in positional hook slots. "State survives" is true only for edits not touching the hook signature; define the migration rule (gap 21). The <1 s claim also assumes style dicts work (gap 19).
- **PreBuildSteps codegen "proven, low-risk" — DOUBTFUL (overclaimed as solved).** UBT gathers the source-file list before pre-build steps run — new/removed generated files need makefile invalidation/project regen; unreal.hx/protoc regenerate *known* file sets (the easy case). Header churn is Live Coding's documented worst case with an active UE 5.8 recompile-loop regression, and "regenerated only on prop-signature change" is exactly what users change constantly. Plus undistributed compiler binaries per build-host OS. Treat the ship path as an engineering problem with named mitigations (gap 14 + stable-file-set codegen design), not a solved precedent.
- **"Coalescing natively gives MVVM Delayed semantics — the mode 76x faster than Immediate" — DOUBTFUL (benchmark misapplication).** The 976.7 ms vs 12.8 ms figure measures batching of *compiled binding writes*; a coalesced ReactiveUI pass re-runs components, allocates a vtree, and diffs — strictly more work per update. Defensible claim: "we batch to one pass per frame like Delayed mode." Whether re-render+diff beats even Immediate bindings at realistic rates is exactly what the missing benchmark plan must establish before any marketing copy.
- **Fab model (MIT core + $29.99–$59.99 tooling, Epic-built binaries, 3-version support) — DOUBTFUL (economically/temporally oversold).** Mechanically compliant, but: one-time revenue vs permanent full-time support burden; forum consensus doubts even €400–500/month; ~$1.2K/yr mean; weeks-to-months review latency vs must-support-latest at ~2 releases/year; the mandatory ~18-month window runs into the UE6 transition (EA end-2027) where Slate carry-over is assumed without evidence; Epic gives away SlateIM/ViewModel/CommonUI, and a free VS Code extension (this family's own precedent) undercuts the paid tooling layer. Plan accordingly (gap 20).
- **`FRUIContext&` makes out-of-render hooks "a compile error" — DOUBTFUL (overclaim).** It only makes hooks uncallable without a context reference in scope; capturing `Ctx` in an `OnClicked` lambda or stored callback compiles cleanly and corrupts the slot cursor at runtime — and the event system encourages exactly such captures. Honest claim: "narrows the error class and removes per-hook TLS lookups"; keep the debug-only thread-local in-render assert, and consider making `FRUIContext` non-copyable/non-capturable-by-value to harden.
