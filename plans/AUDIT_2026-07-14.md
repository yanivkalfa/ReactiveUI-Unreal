### 6b. Factual errors on existing pages (verified against source; several are in the pages
authored 2026-07-14 — recorded here with the same severity as anything else)

1. **Invalid directive samples** — `LanguageReferencePage` FLOW sample and `KeysGuidePage` sample
   put bare markup inside `@if/@for`. The grammar requires `return ( … );` inside directive blocks
   — every demo does it and the formatter corpus pins it ("exploded returns",
   `uetkx-formatter-cases.json`). Both samples would not compile.
2. **Phantom CVars** — `ConceptsPage` + `DebuggingPage` document `rui.Stats` and `rui.DumpTree`.
   Neither exists (`rui.Stats` appears only in a comment, `RuiReconciler.h:65`). The REAL set
   (`RuiCoreMisc.cpp`, the only `TAutoConsoleVariable`s): `rui.TimeSlicing`, `rui.FrameBudgetMs`,
   `rui.HostNodePool`, `rui.HookValidation`, `rui.StrictDiagnostics`, `rui.StrictMode` — plus
   `stat ReactiveUI` (STATGROUP with Renders/Commits/Placements/Updates/Deletions). Note:
   CLAUDE.md's conventions line also cites `rui.Stats`/`rui.DumpTree` as examples — amend there too.
3. **StylingPage denies what exists** — "no USS, no stylesheet cascade" is false. Real system
   (`RuiStyle.h`): inline `style` dict + `classes` (`RegisterStyleClass`, `BuildEffectiveStyle`)
   + theme tokens (`RegisterTheme`, `SetActiveTheme`, `$token`, `LoadStylesheet` parsing
   `@theme`/`.class` source), cascade **theme < classes < inline**; fluent `RUI::Style()` /
   `RUI::Slot()` builders; reserved props `Style/Classes/Ref/SlotProps/MemoEquals/bReuseBySlot`.
   Also the page's `Slot.HAlign="HAlign_Fill"` is an invented literal — accepted values are
   `fill|left|center|right` (`RuiCoreAdapters.cpp:10`). StyledPanels' `classes="rui-demo-dim"`
   demonstrates the classes layer live.
4. **Router & Suspense documented as markup tags** — `RouterPage` shows `<Router>/<Routes>/<Route
   Path>/<Outlet>/<NavLink>`; `SuspensePage` shows `<Suspense Fallback={…}>`. Structural
   primitives are `RUI::` calls, not tags (`new-component` skill; inventory: no such node
   factories; `NavLink` doesn't exist at all — `RUI::Link` does; nesting is the `FRuiRoute` struct
   + `UseOutlet()`; Suspense is `RUI::Suspense(IsReady, Fallback, Children, Key)` — the `IsReady`
   driver is undocumented). Both pages need rewrites around the real API.
5. **"Post-v1 subsystems" page (`FeaturesPage`) is mislabeled** — everything on it SHIPPED in
   Phase 7 (router, stylesheets, ListView/TileView, DnD, presence, CommonUI/MVVM citizenship,
   widget batch 2 — all verified in code). Its own body admits "Everything below is C++ today."
   The title + the "Beyond v1" nav section present shipped features as future. Dissolve into
   current-feature pages.
6. **AssetsPage lifetime advice inverted** — it tells users to keep the resource object alive;
   actually `MakeAssetBrush` brushes are tracked by a process-wide `FGCObject` root
   (`FRuiAssetBrushRoot`, `RuiAssetBrush.cpp:12`) that keeps the `UObject` alive while the brush
   lives. Rewrite to the real contract (+ `NumTrackedAssetBrushes()`).
7. **ComponentsOverviewPage** — lists `Fragment`/`Suspense` as tags (Suspense isn't; fragment is
   the `<>` syntax, distinct from a tag); omits the shipped specials (`ExpandableArea` — incl. its
   `slot.role="header"/"body"` convention — `SegmentedControl`, `NumericEntryBox`, `ComboBox`,
   `SuggestionTextBox`, `ListView`, `TileView`).
8. **Status inconsistency** — README says "Status: **beta** — built end to end"; docs Intro alert

9. **Missing language features** — `{children}` splicing (TD-015, `UetkxCodegen.cpp:1031`),
